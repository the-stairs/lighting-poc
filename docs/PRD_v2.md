# 📌 POC PRD – 기능 우선순위 정리 (광원 크기 & 페더 분리 반영)

## 🎯 1. MVP (최소기능제품)

실제 촬영 현장에서 **즉시 검증 가능한 핵심 기능**만 포함합니다.
👉 **광원 본체 크기**와 \*\*페더(Softness)\*\*를 분리 제어하도록 수정.

| 기능                           | 설명                                                                                                                                                                                                                                     | 난이도 | 우선순위 |
| ------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------ | -------- |
| **배경 색상 설정**             | - 기본값: **검정**<br>- 옵션: **회색**(흰색 제외)<br>- 색상 선택: **팔레트(컬러 피커)**                                                                                                                                                  | 쉬움   | 상       |
| **조명 생성 (클릭 기반)**      | 화면 클릭 시 조명 생성<br>- **모양 선택:** 원형, 사각형 지원                                                                                                                                                                             | 보통   | 상       |
| **조명 선택/이동**             | 생성된 조명을 클릭하면 **선택 상태** 진입<br>- 드래그: 위치 이동                                                                                                                                                                         | 보통   | 상       |
| **조명 속성 제어 (패널 UI)**   | 선택된 조명의 속성을 패널에서 조절:<br>- **크기(Size)**: 광원 본체(핫스폿) 크기<br>- **밝기(Intensity)**: 광원 중심부 밝기 (HDR 스케일)<br>- **페더/소프트니스(Feather/Softness)**: 가장자리 퍼짐 크기<br>- **색상(Color)**: 팔레트 선택 | 보통   | 상       |
| **빛 블렌딩 효과 (강도 조절)** | - Additive Blending 기반<br>- 조명 간 중첩 시 실제 빛처럼 섞임<br>- 블렌딩 강도: 슬라이더 제어                                                                                                                                           | 어려움 | 상       |
| **쉐이더 기반 라이트 렌더링**  | - 프래그먼트 쉐이더 사용<br>- **inverse-square falloff**, **핫스폿+페널브라** 구현<br>- 선형 색 공간 연산 후 sRGB 변환                                                                                                                   | 보통   | 상       |
| **여러 조명 동시 생성**        | 화면에 다수의 조명 배치 및 개별 선택/제어 가능                                                                                                                                                                                           | 보통   | 상       |
| **조명 삭제**                  | 선택된 조명을 **패널 내 삭제 버튼** 또는 **더블탭 제스처**로 제거                                                                                                                                                                        | 쉬움   | 상       |
| **패널 토글 (단축키)**         | 패널을 단축키/버튼으로 숨김·표시 가능<br>- **패널 숨김 시 선택 해제** 처리                                                                                                                                                               | 쉬움   | 상       |

👉 **MVP 목표:**

- 크기와 페더를 독립적으로 조절해 **조명 본체 vs 가장자리 퍼짐을 명확히 제어**
- 실제 조명 같은 핫스폿 + 페더 구조를 POC에서 검증
- Additive Blending + HDR 스케일로 충분히 밝은 광원 연출

| **멀티 디스플레이 확장**     | 5면 디스플레이(벽+천장) 확장 지원, 좌표 매핑 필요                                                     | 어려움 | 중       |
| **좌표 매핑**                | 여러 화면을 합쳤을 때 실제 공간 좌표와 일치하도록 구현                                                | 어려움 | 중       |
| **UI 패널 확장**             | 유지 시간, 프리셋 불러오기 등 추가 속성 제어 지원                                                     | 보통   | 하       |
| **프리셋 저장/로드**         | 자주 쓰는 조명 세팅 저장 후 불러오기                                                                  | 보통   | 하       |
| **HEX/RGB 입력 방식**        | 팔레트 선택 외에 정확한 색상 코드 입력 기능                                                           | 보통   | 하       |
| **디스플레이/컨트롤러 분리** | 촬영 화면(디스플레이)과 제어 UI(컨트롤러)를 분리 운영                                                 | 어려움 | 하       |
| **선택 표시 개선**           | 촬영 화면에서 Border 대신, 패널 내 리스트 UI로 선택 표시                                              | 보통   | 하       |
| **블룸(Post Effect)**        | 밝은 영역만 추출해 블러링 후 원본에 더해, **눈부심 효과** 구현                                        | 보통   | 하       |
| **색온도/프리셋**            | 3200K(텅스텐), 5600K(주광) 등 색온도 기반 조명 프리셋 제공                                            | 보통   | 하       |
| **HDR 모드 지원**            | HDR 디스플레이 환경에서 초고휘도 조명 효과 표현                                                       | 어려움 | 하       |

## 📌 주요 변경점 요약

1. **크기(Size) = 광원 본체(핫스폿) 크기**
2. **페더(Softness) = 가장자리 퍼짐 정도**
3. 기획 의도와 다르게 개발된 부분을 수정 → 크기와 페더를 명확히 분리
4. MVP에서 바로 검증할 수 있게 **패널 슬라이더를 각각 제공**
