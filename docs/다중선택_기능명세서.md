# 레이어 목록 다중 선택 기능 명세서

## 1. 개요

컨트롤 페이지의 **패널 > 레이어 목록**에서 조명을 **Ctrl+클릭**, **Shift+클릭**으로 다중 선택하고 다중 선택 시 캔버스에서 동일 간격 유지 이동, 선택 해제, 복사 및 붙혀넣기 기능을 추가한다. 그리고 추후 속성 일괄 설정 등으로의 확장을 고려한다.

---

## 2. 범위 및 목표

| 항목          | 내용                                                                                                                                      |
| ------------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| **대상 UI**   | 컨트롤 페이지 · 사이드 패널 · 「📚 레이어 목록」 내 조명 리스트(`#layerList`)                                                             |
| **선택 방식** | 일반 클릭(단일), Ctrl+클릭(토글), Shift+클릭(범위)                                                                                        |
| **즉시 구현** | 다중 선택, 캔버스에서 동일 간격 유지 이동, 빈 영역 또는 탭 내 다른 영역 클릭 시 선택 해제, 다중 선택한 요소 복사 및 붙혀넣기(Ctrl + C, V) |
| **확장 고려** | 선택 조명 일괄 속성 설정, 다중 삭제 등                                                                                                    |

---

## 3. 현재 구조 요약 (참고)

- **상태**: `appState.selectedLightId` (단일 ID) → 다중 선택 시 **`selectedLightIds: string[]`** (또는 유사 구조)로 확장 필요.
- **레이어 리스트**: `ui.js`의 `renderLayerList(lights, selectedId)`에서 한 개 `selectedId` 기준으로 `layer-row.selected` 적용.
- **캔버스 선택/드래그**: `main.js`의 `mousePressed`에서 hit 테스트 후 `selectedLightId` 설정, `mouseDragged`에서 해당 한 개 조명만 이동.
- **이벤트**: `app:selected`로 단일 선택 조명(`detail`) 전달 → 다중 선택 시 배열 또는 대표 조명 등 정책 필요.

## 4. 기능 요구사항

### 4.1 레이어 목록에서의 다중 선택

| 구분           | 동작                                                                                                       | 비고                                                |
| -------------- | ---------------------------------------------------------------------------------------------------------- | --------------------------------------------------- |
| **일반 클릭**  | 해당 행만 선택(기존 단일 선택과 동일). 기존 다중 선택 해제 후 해당 조명 1개만 선택.                        | `selectedLightIds` → `[id]` 또는 primary만 설정     |
| **Ctrl+클릭**  | 클릭한 행을 선택 집합에 **토글**. 이미 선택된 행이면 선택 해제, 아니면 선택 추가.                          | 선택 순서 유지 권장(붙여넣기/순서 의존 시 활용)     |
| **Shift+클릭** | 리스트 상 **마지막 선택(또는 primary)부터 클릭한 행까지** 범위로 선택. 그 사이 모든 행이 선택 집합에 포함. | 리스트 인덱스 기준 범위 (현재 렌더 순서: 역순 주의) |

**제약/정책**

- 최소 1개는 선택 유지할지, 0개(전체 해제) 허용할지 정책 결정 필요.  
  → **권장**: 빈 영역 클릭 시 0개 허용; 레이어 목록에서는 Ctrl/Shift로만 조작 시 최소 1개 유지 가능.
- 드래그로 순서 변경 시, 다중 선택 집합은 **ID 기준**이므로 재정렬 후에도 동일 ID 집합 유지.

### 4.2 캔버스에서의 동작

#### 4.2.1 클릭으로 선택 변경

| 상황                         | 동작                                                    |
| ---------------------------- | ------------------------------------------------------- |
| **캔버스에서 조명 1개 클릭** | 해당 조명만 선택(단일 선택). 기존 다중 선택 해제.       |
| **캔버스에서 빈 영역 클릭**  | **다중 선택 해제**(선택 0개). 기존 단일/다중 모두 해제. |

(추후) 캔버스에서도 Ctrl+클릭 / Shift+클릭으로 다중 선택 확장 시, 위 표와 동일한 의미로 확장 가능하도록 이벤트/상태 구조를 두는 것을 권장.

#### 4.2.2 다중 선택된 조명의 동시 이동(동일 간격 유지)

| 항목          | 내용                                                                                                                                                                                                 |
| ------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **트리거**    | 다중 선택된 조명 중 **어느 하나**를 캔버스에서 드래그 시작.                                                                                                                                          |
| **동작**      | 선택된 **모든 조명**이 함께 이동하며, **서로 간의 상대 위치(간격·각도)는 유지**.                                                                                                                     |
| **구현 요약** | 드래그 시작 시: (1) 드래그 대상 조명 기준으로 나머지 선택 조명들의 **상대 벡터(dx, dy)** 저장. (2) 드래그 중: 마우스 delta만큼 **전체 선택 집합**을 이동시키고, 각 조명은 저장된 상대 벡터대로 배치. |

**세부 규칙**

- 드래그 오프셋은 **드래그를 시작한 그 조명** 기준으로만 계산 (기존 단일 드래그와 동일).
- 회전/스케일은 본 명세 범위 외; “동일 간격 유지”는 **평행 이동**만 다룸.

### 4.3 단일 선택 복사/붙여넣기 (현재 구현 상태)

현재 에디터에는 **단일 선택 조명 기준 Ctrl + C / Ctrl + V 복사·붙여넣기**가 이미 구현되어 있으며, 다중 선택 기능은 이 동작을 **확장**하는 방향으로 설계한다.

| 항목             | 내용                                                                                                  |
| ---------------- | ----------------------------------------------------------------------------------------------------- |
| **대상 선택**    | 현재는 `selectedLightId`로 표현되는 **단일 선택 조명 1개**만 복사 대상                               |
| **Ctrl + C**     | 선택된 조명이 있을 때, 해당 조명의 주요 속성(위치/크기/색/밝기/페더/각도/타이머 등)을 내부 클립보드에 저장 |
| **Ctrl + V**     | 내부 클립보드에 저장된 조명을 기준으로, 약간 옆으로 오프셋된 위치에 **새 ID를 가진 조명**을 생성해 붙여넣기 |
| **선택 상태**    | 붙여넣기 직후 **새로 생성된 조명 1개가 선택**되며, 패널·레이어 리스트가 이에 맞춰 갱신                 |
| **Shoot 모드**   | `mode === "shoot"` 인 경우에는 Ctrl + C / V 단축키가 동작하지 않음                                   |
| **입력 포커스**  | `INPUT`, `TEXTAREA`, `SELECT` 등에 포커스가 있을 때는 브라우저 기본 텍스트 복사/붙여넣기를 우선시      |

다중 선택 기능 구현 시에는, 위 단일 선택 복사/붙여넣기 로직을 아래와 같이 확장하는 것을 목표로 한다.

- 복사 대상: `selectedLightIds` 전체 (또는 primary 기준 상대 위치 포함)
- 붙여넣기 결과: 여러 개의 새 ID를 가진 조명 집합이 생성되고, **선택 집합이 새 조명들로 전환**되는 형태

### 4.3 선택 해제

| 트리거                                | 동작                                                                |
| ------------------------------------- | ------------------------------------------------------------------- |
| **캔버스 빈 영역 클릭**               | 다중 선택 전부 해제.                                                |
| **(선택) 패널 밖 클릭 / 포커스 아웃** | 정책에 따라 해제 여부 결정. 권장: 캔버스 빈 영역만 명시적으로 해제. |

## 5. 확장성 설계

추후 아래 기능을 추가할 때 상태/API를 바꾸지 않고 확장할 수 있도록 한다.

### 5.1 상태 구조 제안

- **`selectedLightIds: string[]`**  
  선택된 조명 ID 배열. 순서 유지(리스트 순 또는 마지막 클릭 순).
- **(선택) `primarySelectedId: string | null`**  
  “대표” 선택 1개. 패널에서 속성 편집 시 이 조명 기준 표시, 복사/붙여넣기 시 기준점 등.
  - 단일 선택 시: `selectedLightIds.length === 1`이고 `primarySelectedId`가 그 ID.
  - 다중 선택 시: 마지막으로 (또는 Shift 범위의 끝으로) 선택된 ID를 primary로 두는 방식 등.

기존 `selectedLightId`와의 호환은 다음 중 하나로 처리 가능.

- **호환 모드**: `getSelectedLight()`가 `primarySelectedId` 또는 `selectedLightIds[0]` 반환하도록 하고, 기존 단일 선택 UI/이벤트는 그대로 사용.
- **점진적 전환**: `selectedLightId` deprecated 후 `selectedLightIds` + `primarySelectedId`만 사용.

### 5.2 이벤트 제안

- **`app:selected`**
  - 단일 선택: `detail: Light | null` (기존 유지).
  - 다중 선택 대응: `detail.selectedLights: Light[]`, `detail.primary: Light | null` 등으로 확장하거나, 별도 `app:selectionChanged` 이벤트 추가.
- **`app:selectionCleared`**  
  빈 영역 클릭 등으로 선택이 0개가 될 때 발생. (선택)

### 5.3 확장 기능과의 연동

| 확장 기능                         | 연동 포인트                                                                                                                                                |
| --------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **복사/붙여넣기**                 | (현재) 단일 선택: `selectedLightId` 기준 복사/붙여넣기 구현 완료. (향후) `selectedLightIds`(또는 `primarySelectedId` 기준 상대 위치)로 복사 대상을 확장. 붙여넣기 시 새 ID 생성 후 `lights`에 추가, 선택을 새로 붙여넣은 ID들로 갱신. |
| **속성 동시 설정**                | 패널에서 값 변경 시 `getSelectedLights()`로 선택 조명 배열을 가져와, 동일 속성(opacity, 색, 크기 등)을 일괄 적용.                                          |
| **다중 삭제**                     | “선택 조명 삭제” 시 `selectedLightIds`에 있는 모든 ID를 `lights`에서 제거 후 `selectedLightIds` 비우기.                                                    |
| **Bring to Front / Send to Back** | 선택된 ID들 전체를 한 덩어리로 앞/뒤로 재정렬.                                                                                                             |

위를 위해 **공개 API**에 다음이 있으면 유리하다.

- `getSelectedLights(): Light[]` — 선택된 조명 객체 배열.
- `setSelection(ids: string[])` — 선택 집합 설정.
- `addToSelection(id: string)`, `removeFromSelection(id: string)`, `toggleSelection(id: string)` — 선택 집합 조작.
- `clearSelection()` — 기존 유지, 다중 선택 전부 해제.

## 6. UI/UX 요구사항

### 6.1 레이어 목록

- 다중 선택된 행은 **모두** 시각적으로 강조 (예: `layer-row.selected` 또는 `layer-row.multi-selected`).
- (선택) primary 선택 행만 두껍게/다른 색으로 구분.
- 툴팁/안내: “Ctrl+클릭: 다중 선택, Shift+클릭: 범위 선택” 등.

### 6.2 캔버스

- 다중 선택 시 **선택된 모든 조명**에 선택 하이라이트(테두리 등) 표시.
- 드래그 시 이동되는 조명이 여러 개임을 직관적으로 알 수 있도록 (예: 함께 움직이는 것처럼 보이기).

### 6.3 패널

- 단일 선택 시: 기존처럼 해당 조명 속성만 표시.
- 다중 선택 시:
  - “N개 선택됨” 표시,
  - (확장) “속성 일괄 적용” UI 또는 일부 슬라이더를 “선택된 모든 조명에 적용”으로 동작.

## 7. 구현 시 유의사항

1. **레이어 리스트 렌더 순서**  
   `renderLayerList`가 `lights`를 **역순**으로 그리므로, Shift+클릭 범위 계산 시 **배열 인덱스**와 **DOM 순서** 매핑을 일치시키기 (예: 화면 상 첫 행 = 배열의 마지막 요소).
2. **드래그 시작 판별**  
   다중 선택 상태에서 캔버스 mousePressed 시, hit 테스트 결과가 **선택 집합에 포함된 조명**이면 “그 조명을 기준으로 전체 이동”으로 진입.
3. **기존 단일 선택 호환**  
   `selectLightById(id)`는 “해당 1개만 선택”으로 동작하도록 유지하고, `setSelection([id])` 또는 내부적으로 `selectedLightIds = [id]`로 통일하면 호환 유지에 유리함.

## 8. 체크리스트 (구현 검증용)

- [ ] 레이어 목록에서 일반 클릭 → 해당 조명만 선택, 다중 선택 해제.
- [ ] 레이어 목록에서 Ctrl+클릭 → 해당 행 선택 토글 (추가/제거).
- [ ] 레이어 목록에서 Shift+클릭 → 마지막 선택(또는 primary)부터 클릭한 행까지 범위 선택.
- [ ] 캔버스 빈 영역 클릭 → 선택 전부 해제.
- [ ] 캔버스에서 다중 선택된 조명 중 하나 드래그 → 모든 선택 조명이 동일 간격 유지하며 이동.
- [ ] 다중 선택 시 레이어 목록에서 여러 행이 선택 스타일로 표시됨.
- [ ] 다중 선택 시 캔버스에서 모든 선택 조명에 하이라이트 표시됨.
- [ ] `getSelectedLights()`, `setSelection(ids)`, `clearSelection()` 등 확장용 API 설계 반영 여부.
